🧭 Trip Manager – Deployment & Documentation

🚀 Deployment Overview

This application is deployed on Heroku (for backend) and Vercel (for frontend). It allows users to plan trips with preferences like avoiding tolls or highways, and stores trip data using a simple JSON-based backend.

  • Backend: Node.js + Express.js
  • Frontend: HTML + JavaScript + Google Maps API
  • Data Store: Local JSON file (trips.json)

🛠️ Setup Instructions

Backend (Heroku)


    git clone https://github.com/yourusername/trip-manager
    cd trip-manager
    npm install
    heroku login
    heroku create trip-manager-app
    git push heroku main
    heroku open

Frontend (Vercel)


    vercel login
    vercel init
    vercel deploy

📡 API Documentation

Base URL: https://trip-manager-app.herokuapp.com/api/trips

GET /api/trips

Returns all saved trips.

GET /api/trips/:id

Returns a specific trip by ID.

POST /api/trips

Creates a new trip.

{
  "origin": "Pune",
  "destination": "Nashik",
  "avoidTolls": true,
  "avoidHighways": false,
  "date": "2025-09-16"
}

PUT /api/trips/:id

Updates an existing trip.

👤 User Guide

  1. Open the frontend app on Vercel.
  2. Enter your origin and destination.
  3. Select preferences (avoid tolls/highways).
  4. Click "Get Route" to view your trip on the map.
  5. Click "Save Trip" to store it in the backend.
